    public assign(target: T, source: S): T & S source 

    import {assign} from 'kogu/src/oop/assign.ts'

    Super simplified version of Object.assign which all it does is take the enumerable "own properties" (using a for in loop) from a source object and assign the key-value pairs to a target object using bracket notation. It can also be used to create a shallow copy of an object with assign({}, obj).

    Params:

    NameTypeAttributeDescription
    target T

    Object that will receive key-value pairs.
    source S

    Object that will provide the key value-pairs.

    Return:

    T & S

    A union of objects.

    Example:

    const source = { a: 5, b: { c: 1 } };

assign({}, source) === source; //> false
assign({}, source).b === source.b; // true
assign({ d: true }, source);
//> { d: true, a: 5, b: { c: 1 } }
assign(source, { d: "D" });
//> { d: "D", a: 5, b: { c: 1 } }
//* Note: [source] mutated.

    See:

    • https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/Object/assign

    public assoc(key: K extends keyof any, value: V): function(obj: T): T & Record<K, V> source 

    import {assoc} from 'kogu/src/oop/assoc.ts'

    Takes a key-value pair as separate arguments and then returns a setter function that mutates the provided object by either adding a new key to it and setting its value or changing the current value of that key if it exists (mutates the object).

    This function preserves the prototype of the object and works well with arrays which can be used in place of arr[idx] = ?.

    Params:

    NameTypeAttributeDescription
    key K extends keyof any

    Key to replace or insert.
    value V

    Associated value to the key.

    Return:

    function(obj: T): T & Record<K, V>

    A setter function that adds or changes a key-value pair.

    Example:

    const author = new Person("Stephen King", 72);

assoc("age", 25)(author);
//> Person { name: "Stephen King", age: 25 }
assoc("books", ["It"])(author);
//> Person { name: "Stephen King"
//>        , books: ["It"]
//>        , age: 25 }
assoc(0, -5)([5, 6, 7]); //> [-5, 6, 7]

    public between(lower: number, upper: number): function(n: number): number source 

    import {between} from 'kogu/src/math/between.ts'

    Returns a function that keeps a number from being lower or higher than the specified boundaries. This function is also called clamp in other APIs and utility libraries. If you only need to contain lower and upper limits separately, use the partially applied versions of max and min instead.

    Note: The order of the two arguments is merely symbolic, they will be flipped if lower is greater than upper.

    Params:

    NameTypeAttributeDescription
    lower number

    Lower limit.
    upper number

    Upper limit.

    Return:

    function(n: number): number

    Function that returns lower if n < lower, upper if n > upper or just n otherwise.

    Example:

    between(0, 10)(12); //> 10
between(-10, 0)(-15); //> -10
between(-1, 1)(0); //> 0
between(5, 5)(5); //> 5

    public caselessEq(a: string, b: string, alpha: boolean): boolean source 

    import {caselessEq} from 'kogu/src/text/case.ts'

    Case-insensitive equality test for two strings. If the option for ignoring non-alphanumeric characters is enabled, then RegExp will remove them from both strings in order to perform a more lenient case-insensitive comparison, but this will also ignore characters like ä, á, etc.

    NameTypeAttributeDescription

    b string

    • optional
    • default: false

    Ignore non-alphanumerics.

    Return:

    boolean

    Result of case-insensitive comparison between two strings.

    public comp(funcs: ...function): function source 

    import {comp} from 'kogu/src/lambda/compose.ts'

    public foldl(reducer: function(acc: R, x: T, i: number): R, initial: R): function(xs: T[]): R source 

    import {foldl} from 'kogu/src/list/foldl.ts'

    Folds a list from left to right using an initial value to accumulate an eventual result. It's similar to reduce although the initial value is obligatory.

    The reducer is a tertiary function whose first argument is the accumulated value, the second argument is the current iterated element, and lastly, the index. The return value of this reducer must be of the same type as the accumulated value.

    Params:

    NameTypeAttributeDescription
    reducer function(acc: R, x: T, i: number): R

    Tertiary reducer function.
    initial R

    Initial value for accumulation.

    Return:

    function(xs: T[]): R

    A function that accepts the array.

    Example:

    foldl(add, 50)(range(1, 3)); //> 56

foldl<string, number>(
 (acc, _, i) => acc + i,
 "Indexes: "
)(repeatedly(noop, 5));
//> "Indexes: 01234"

    public identity(value: T): T source 

    import {identity} from 'kogu/src/lambda/identity.ts'

    Returns the first argument passed to it and ignores the rest.

    It may not seem useful at first, but you can use it in a pipeline or think of it as the absolute unit of function composition. For example, you can implement compose in the Haskell language using id: compose = foldl (.) id since (.) is an infix binary function (takes only two arguments).

    Params:

    NameTypeAttributeDescription
    value T

    The value to return.

    Return:

    T

    The value that you passed.

    Example:

    const refObj = { prop: "Value" };

filter(identity)([true, false]) //> [true]
identity(refObj) === refObj //> true
identity(5) //> 5

    public isEmpty(obj: any): boolean source 

    import {isEmpty} from 'kogu/src/oop/isEmpty.ts'

    Note: The justification for checking the key amount is to support objects as well as arrays, and this edge case: Array(5) doesn't have keys. Also, primitives, undefined, null, NaN and anything that doesn't have keys will return true.

    Params:

    NameTypeAttributeDescription
    obj any

    Object that may be empty.

    Return:

    boolean

    Result of the emptiness test.

    Example:

    isEmpty([1, 2, 3]); //> false
isEmpty([]); //> true
isEmpty("Not empty"); //> false
isEmpty(""); //> true
isEmpty({}); //> true
